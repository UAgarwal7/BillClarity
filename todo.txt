 PHASE 1 — BACKEND ROUTER WIRING (Wire services into endpoints)
================================================================================

  --- bills.py (backend/routers/bills.py) ---

  [ ] 1a. POST /api/bills/upload
      - Call handle_upload() from upload_service
      - Call run_parsing_pipeline() via BackgroundTasks (local mode)
      - Validate file type (PDF, PNG, JPG, HEIC only)
      - Validate file size (max 10 MB each)
      - Return { bill_id, status: "processing" }

  [ ] 1b. GET /api/bills/:bill_id
      - bills_repo.get_by_id(bill_id)
      - 404 if not found
      - Return full bill document

  [ ] 1c. GET /api/bills/:bill_id/line-items
      - line_items_repo.get_by_bill_id(bill_id)
      - Return { line_items: [...], total_count: N }

  [ ] 1d. POST /api/bills/:bill_id/confirm-fields
      - Parse field_corrections from request body
      - Update each line item in MongoDB
      - If code was corrected, re-run benchmark for that item

  --- analysis.py (backend/routers/analysis.py) ---

  [ ] 1e. GET /api/bills/:bill_id/explanation
      - Fetch bill.plain_language_summary from MongoDB
      - If not yet generated, trigger generate_explanation() on demand
      - Return { overall_summary, line_item_explanations }

  [ ] 1f. GET /api/bills/:bill_id/errors
      - Aggregate all flags from line_items where flags array is non-empty
      - Count by severity (critical, warning, info)
      - Return { error_summary, errors, cross_document_issues }

  [ ] 1g. GET /api/bills/:bill_id/benchmarks
      - Fetch benchmark_results by bill_id
      - Calculate summary: items_above_typical, estimated_savings_low/high
      - Return { summary, benchmarks }

  [ ] 1h. GET /api/bills/:bill_id/insurance-insights
      - Return bill.insurance_insights (stored during pipeline)
      - If not available, trigger match_insurance_rules() on demand

  --- appeal_packets.py (backend/routers/appeal_packets.py) ---

  [ ] 1i. POST /api/bills/:bill_id/appeal-packet/generate
      - Parse sections list from request body
      - Fetch bill, line items, benchmarks, insurance insights
      - Generate each selected section via appeal_service.generate_packet()
      - Store in appeal_packets collection
      - Return { packet_id, status: "generating" }

  [ ] 1j. GET /api/appeal-packets/:packet_id
      - appeal_packets_repo.get_by_id(packet_id)
      - Return full packet with all sections as markdown

  [ ] 1k. PUT /api/appeal-packets/:packet_id
      - Parse updated sections from body
      - appeal_packets_repo.update() with new section content

  [ ] 1l. GET /api/appeal-packets/:packet_id/pdf
      - Fetch packet, call pdf_service.generate_pdf(sections)
      - Upload PDF to S3
      - Return presigned URL or stream file

  --- calls.py (backend/routers/calls.py) ---

  [ ] 1m. POST /api/calls/start
      - Fetch bill + all analysis data
      - Call call_service.start_call_session() → strategy + opening_script
      - Create call_log in MongoDB
      - Return { call_id, strategy, opening_script, key_points }

  [ ] 1n. WS /api/calls/:call_id/stream
      - Accept WebSocket
      - On each received message: append transcript, call process_transcript()
      - Send back { type: "ai_response", text, strategic_note }
      - On disconnect: clean up session

  [ ] 1o. POST /api/calls/:call_id/end
      - Generate call summary via Gemini
      - Update call_log with outcome, summary, next_steps
      - Return { summary, outcome, next_steps }

  [ ] 1p. GET /api/calls/:call_id
      - call_logs_repo.get_by_id(call_id)
      - Return full call log
